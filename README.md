# CppLabsPortfolio

Портфолио лабораторных работ по C++ (каждая работа находится в отдельной папке `Lab*`).

## Лабораторные работы

### [Lab1](Lab1/README.md) — Анализ access.log
- Условие: консольная утилита для анализа `access.log` — вывод запросов со статусом `5xx`, подсчёт самых частых `5xx`-запросов, поиск временного окна с максимумом запросов (CLI-опции `-o/-p/-s/-w/-f/--from/-e/--to`).
- файловый ввод/вывод, парсинг строк, обработка аргументов командной строки, структуры, `std::vector` + алгоритмы (`sort`).
- Подробности: [`Lab1/README.md`](Lab1/README.md).

### [Lab2](Lab2/README.md) — ITMO Endian / `uint239_t`
- Условие: реализовать беззнаковый целочисленный тип `uint239_t` (35 байт, хранение “ITMO Endian”) с конвертациями и арифметикой (`+`, `-`, `*`, `/`), сравнением, выводом в поток и работой со “shift”.
- побитовые операции и работа с байтами, перегрузка операторов, `static_assert`, шаблоны (в т.ч. свой контейнер), модульная структура (`lib/`, тесты).
- Подробности: [`Lab2/README.md`](Lab2/README.md).

### [Lab3](Lab3/README.md) — Модель песчаной кучи + BMP
- Условие: реализовать упрощённую модель Abelian Sandpile; стартовое состояние — из TSV; в процессе симуляции сетка может расширяться; сохранять состояние в BMP с заданной частотой.
- динамическая память и ручное управление ресурсами (RAII через деструктор), работа с файлами (в т.ч. бинарный BMP), структуры/классы, реализация расширяемой 2D-сетки.
- Подробности: [`Lab3/README.md`](Lab3/README.md).

### [Lab4](Lab4/README.md) — Парсер аргументов командной строки
- Условие: спроектировать и реализовать класс `ArgParser`, удовлетворяющий тестам (строковые/целочисленные аргументы, флаги, значения по умолчанию, multi-value, positional, help).
-  ООП (классы, методы), контейнеры STL (`std::map`, `std::vector`), `std::initializer_list`, работа с тестами (GoogleTest).
- Подробности: [`Lab4/README.md`](Lab4/README.md).

### [Lab5](Lab5/README.md) — Бот для “Морского боя” (master/slave)
- Условие: консольное приложение-бот для игры в морской бой со взаимодействием через stdin/stdout (режимы master/slave, команды управления игрой, обмен выстрелами и результатами).
-  ООП и полиморфизм (стратегии стрельбы через `virtual`), разбор команд, генерация случайных чисел, контейнеры STL, сохранение/загрузка состояния в файл.
- Подробности: [`Lab5/README.md`](Lab5/README.md).

### [Lab6](Lab6/README.md) — Поиск маршрутов через API (Yandex Расписания)
- Условие: консольное приложение, которое по дате и списку городов из `cities.txt` запрашивает варианты маршрутов через Yandex API и строит оптимальный маршрут, проходящий через все города (старт — первый город, финиш — последний; в текущей версии запросы идут без пересадок).
-  HTTP-запросы (cpr), парсинг JSON (nlohmann/json), кэширование результатов на диске (json), DP по битмаске (TSP-подобная оптимизация маршрута), работа с UTF‑8 выводом в Windows.
- Подробности: [`Lab6/README.md`](Lab6/README.md).

### [Lab7](Lab7/README.md) — Unrolled Linked List (STL-совместимый контейнер)
- Условие: реализовать STL-совместимый контейнер `unrolled_list` (итераторы, поддержка аллокатора, требования контейнеров/последовательностей, гарантии исключений) + тесты.
-  шаблоны, собственные итераторы (bidirectional), аллокаторы (`std::allocator_traits`, `rebind`), placement new и `aligned_storage`, RAII и гарантия исключений.
- Подробности: [`Lab7/README.md`](Lab7/README.md).

### [Lab8](Lab8/README.md) — Библиотека адаптеров (dataflow/pipeline)
- Условие: реализовать набор адаптеров для “ленивых” пайплайнов обработки данных (по мотивам `std::ranges`): `Dir`, `OpenFiles`, `Split`, `Transform`, `Filter`, `AggregateByKey`, `Join`, `Out`, и др.
-  обобщённое программирование, перегрузка `operator|`, view/итераторы, `std::filesystem`, `std::optional`, `std::expected`, `std::format`, функциональные объекты/лямбды.
- Подробности: [`Lab8/README.md`](Lab8/README.md).

### [Lab9](Lab9/README.md) — Шедулер задач (граф вычислений)
- Условие: реализовать `TTaskScheduler`, который строит граф вычислений, поддерживает “future result” как зависимости, и выполняет задачи лениво/по требованию.
- variadic templates и perfect forwarding, указатели на методы, type erasure (собственный `Any`), `std::shared_ptr`, RTTI (`dynamic_cast`), лямбды.
- Подробности: [`Lab9/README.md`](Lab9/README.md).

### [Lab10](Lab10/README.md) — Интерпретатор ITMOScript
- Условие: реализовать интерпретатор языка ITMOScript (лексер/парсер, вычисление выражений, управляющие конструкции, типы данных, функции, стандартная библиотека; тестируемая функция “вход→выход”).
- разбор текста (lexer/parser), AST и интерпретация, `std::variant` для динамических значений, области видимости/замыкания, исключения для управления потоком, умные указатели.
- Подробности: [`Lab10/README.md`](Lab10/README.md).
